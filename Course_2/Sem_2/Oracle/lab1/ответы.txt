1.Параметры подключения к серверу Oracle включают:

Имя сервера или IP-адрес
Номер порта
Имя базы данных
Имя пользователя и пароль
Опциональные параметры, такие как параметры безопасности или параметры сети.

2.DML операторы (Data Manipulation Language) в Oracle используются для изменения данных в таблицах. Особенности выполнения DML операторов в Oracle включают:

Транзакционность: изменения в таблицах могут быть выполнены в рамках транзакции, которая может быть отменена или подтверждена.
Коммит: изменения могут быть зафиксированы в базе данных с помощью команды COMMIT, что гарантирует сохранение изменений после завершения транзакции.
Rollback: изменения могут быть отменены с помощью команды ROLLBACK, что откатывает транзакцию до начального состояния.
Блокировка: при выполнении DML операторов в таблицах могут быть установлены блокировки, которые могут предотвратить изменение данных другими пользователей.

3.В окне подключения к базе данных Oracle можно увидеть следующие типы объектов:

Таблицы: объекты, в которых хранятся данные в структурированном формате.
Представления: объекты, которые представляют данные из одной или нескольких таблиц в виде виртуальной таблицы.
Процедуры и функции: объекты, которые содержат код, выполняющий определенные действия на базе данных.
Индексы: объекты, которые ускоряют поиск данных в таблицах.
Синонимы: объекты, которые предоставляют альтернативные имена для других объектов в базе данных.

4.Некоторые известные типы данных в Oracle включают:

Числа (NUMBER)
Строки (VARCHAR2, CHAR)
Дата и время (DATE, TIMESTAMP)
Бинарные данные (BLOB, CLOB)
Логические значения (BOOLEAN)
XML-документы (XMLTYPE)

5.Некоторые известные ограничения целостности в Oracle включают:

Ограничения на уникальность (UNIQUE)
Ограничения на внешние ключи (FOREIGN KEY)
Ограничения на проверку значений (CHECK)
Ограничения на NOT NULL

RTO - Время восстановления - recovery time objective - допустимое время простоя сервиса в случае сбоя
RPO - Точка возврата - recovery point objective  - допустимый объем возможных потерь данных в случае сбоя
SYS – предопределенный привилегированный пользователь ранга администратора базы данных, который является владельцем ключевых ресурсов БД Oracle 
SYSTEM – предопределенный привилегированный пользователь, которому принадлежат ключевые ресурсы БД Oracle 
DBA – предопределенная роль, которая автоматически создаётся для каждой базы данных Oracle и содержит все системные привилегии, кроме SYSDBA и SYSOPER
SYSDBA и SYSOPER - специальные привилегии администратора, которые позволяют выполнять базовые задачи администрирования: запуск или остановка экземпляра БД; создание, удаление, открытие или монтирования базы и др. 
 Роль DBA не включает SYSDBA и SYSOPER. Привилегии могут быть указаны при подключении (connect) пользователя к БД
